🔍 Chat System Debug Guide

This guide helps you debug the chat leads system step by step.

📋 Step 1: Check Database Tables

Run this SQL in your Supabase SQL Editor:

-- Check if chat_sessions table exists and has data SELECT COUNT(*) as session_count FROM chat_sessions; SELECT * FROM chat_sessions ORDER BY created_at DESC LIMIT 5; -- Check if chat_leads table exists and has data SELECT COUNT(*) as lead_count FROM chat_leads; SELECT * FROM chat_leads ORDER BY created_at DESC LIMIT 5; -- Check if chat_messages table exists and has data SELECT COUNT(*) as message_count FROM chat_messages; SELECT * FROM chat_messages ORDER BY created_at DESC LIMIT 10;

🧪 Step 2: Test Session Creation

Create a test session manually:

-- Create a test session INSERT INTO chat_sessions (session_id, user_email, is_active) VALUES ('test_session_' || extract(epoch from now()), 'test@example.com', true); -- Check if it was created SELECT * FROM chat_sessions WHERE session_id LIKE 'test_session_%';

💬 Step 3: Test Message Saving

Add test messages:

-- Find your test session ID first SELECT session_id FROM chat_sessions WHERE session_id LIKE 'test_session_%' ORDER BY created_at DESC LIMIT 1; -- Replace 'your_session_id' with the actual session ID from above INSERT INTO chat_messages (session_id, message, sender_type) VALUES ('your_session_id', 'Hello, I need help with a custom suit', 'user'), ('your_session_id', 'I can help you with that! What type of suit are you looking for?', 'assistant'), ('your_session_id', 'Something formal for a wedding', 'user'); -- Check messages were saved SELECT * FROM chat_messages WHERE session_id = 'your_session_id';

🎯 Step 4: Test Edge Function

Test the process-chat-lead function:

-- In your browser console (F12), run this JavaScript: fetch('/functions/v1/process-chat-lead', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer YOUR_SUPABASE_ANON_KEY' }, body: JSON.stringify({ sessionId: 'your_session_id' }) }) .then(response => response.json()) .then(data => console.log('Edge function result:', data)) .catch(error => console.error('Edge function error:', error));

🔧 Step 5: Check Environment Variables

Verify your environment variables are set:

# Check if your .env file exists and contains: VITE_SUPABASE_URL=your_supabase_url VITE_SUPABASE_ANON_KEY=your_supabase_anon_key VITE_OPENAI_API_KEY=your_openai_api_key
⚠️ Important: Make sure you have a .env file in your project root with these variables.

🔍 Step 6: Real-World Test

Test the actual chatbot:

  1. Start your development server: npm run dev
  2. Open your website in a browser
  3. Click the chat button (should be in bottom-right corner)
  4. Fill out the information form (name, email, phone)
  5. Send a few messages
  6. Close the chat
  7. Check the admin panel: /admin/chat-leads

🐛 Common Issues & Solutions

❌ Problem: Database tables don't exist

Solution: Run the migration again in Supabase SQL Editor:

-- Run this in Supabase SQL Editor ALTER TABLE public.chat_sessions ADD COLUMN IF NOT EXISTS user_name VARCHAR(255), ADD COLUMN IF NOT EXISTS user_phone VARCHAR(50), ADD COLUMN IF NOT EXISTS user_info_collected BOOLEAN DEFAULT false, ADD COLUMN IF NOT EXISTS updated_at TIMESTAMPTZ DEFAULT NOW(); ALTER TABLE public.chat_leads ADD COLUMN IF NOT EXISTS customer_phone VARCHAR(50);

❌ Problem: Edge function not working

Solution: Check the edge function logs in Supabase Dashboard > Edge Functions

❌ Problem: Chat button not appearing

Solution: Check browser console for JavaScript errors

❌ Problem: Environment variables not loading

Solution: Restart your development server after adding .env file

🧹 Step 7: Cleanup Test Data

Remove test data when done:

-- Clean up test sessions and related data DELETE FROM chat_leads WHERE session_id LIKE 'test_session_%'; DELETE FROM chat_messages WHERE session_id LIKE 'test_session_%'; DELETE FROM chat_sessions WHERE session_id LIKE 'test_session_%';

✅ Success Indicators:

  • Chat sessions are being created in the database
  • Messages are being saved to chat_messages table
  • Edge function processes leads successfully
  • Chat leads appear in the admin panel
  • User information is collected and stored
