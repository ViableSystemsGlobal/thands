🔧 Admin Debug Tool

This tool helps debug the admin panel database connection and data issues.

📋 Step 1: Run Database Checks

Copy and paste these SQL queries into your Supabase SQL Editor:

🔍 Check Table Structure:

-- Check if chat tables exist and their structure SELECT table_name, column_name, data_type, is_nullable FROM information_schema.columns WHERE table_name IN ('chat_sessions', 'chat_leads', 'chat_messages') ORDER BY table_name, ordinal_position;

📊 Check Data Counts:

-- Count records in each table SELECT 'chat_sessions' as table_name, COUNT(*) as record_count FROM chat_sessions UNION ALL SELECT 'chat_leads' as table_name, COUNT(*) as record_count FROM chat_leads UNION ALL SELECT 'chat_messages' as table_name, COUNT(*) as record_count FROM chat_messages UNION ALL SELECT 'chat_rate_limits' as table_name, COUNT(*) as record_count FROM chat_rate_limits UNION ALL SELECT 'chat_abuse_reports' as table_name, COUNT(*) as record_count FROM chat_abuse_reports;

🔎 Check Recent Data:

-- Check recent chat sessions SELECT id, session_id, user_email, user_name, user_phone, user_info_collected, created_at FROM chat_sessions ORDER BY created_at DESC LIMIT 10; -- Check recent chat leads SELECT id, customer_name, customer_email, customer_phone, status, message_count, created_at FROM chat_leads ORDER BY created_at DESC LIMIT 10; -- Check recent chat messages SELECT cm.id, cm.message, cm.sender_type, cm.created_at, cs.user_email, cs.user_name FROM chat_messages cm LEFT JOIN chat_sessions cs ON cm.session_id = cs.id ORDER BY cm.created_at DESC LIMIT 10;

🔐 Step 3: Check Row Level Security (RLS)

Check if RLS policies are blocking admin access:

-- Check RLS policies for chat tables SELECT schemaname, tablename, policyname, permissive, roles, cmd, qual FROM pg_policies WHERE tablename IN ('chat_sessions', 'chat_leads', 'chat_messages', 'chat_rate_limits', 'chat_abuse_reports') ORDER BY tablename, policyname; -- Check if tables have RLS enabled SELECT schemaname, tablename, rowsecurity FROM pg_tables WHERE tablename IN ('chat_sessions', 'chat_leads', 'chat_messages', 'chat_rate_limits', 'chat_abuse_reports');

⚠️ Common Issues & Solutions

❌ Issue: Tables don't exist

Solution: Run the migrations again. The tables might not have been created properly.

❌ Issue: No data in tables

Solution: The chat system hasn't been used yet, or it's not working properly. Create test data using Step 2.

❌ Issue: RLS blocking access

Solution: Make sure your admin user has the 'admin' role in the profiles table.

❌ Issue: Admin user not recognized

Solution: Check that you're logged in as an admin and your profile has role = 'admin'.

✅ Success Checklist

  • All chat tables exist with proper structure
  • Tables contain data (either real or test data)
  • RLS policies allow admin access
  • Admin user has proper role in profiles table
  • Admin pages load without JavaScript errors
  • Edge functions are working (if testing chat flow)
